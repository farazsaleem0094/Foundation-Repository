# Versioning Strategy

The repository follows Semantic Versioning (SemVer).

Format:

```
MAJOR.MINOR.PATCH
```

## MAJOR

Increment when incompatible or breaking changes are introduced.

Example:

```
1.0.0 → 2.0.0
```

---

## MINOR

Increment when new capabilities are added without breaking existing functionality.

Example:

```
1.1.0 → 1.2.0
```

---

## PATCH

Increment for corrections, clarifications, typo fixes, or non-breaking improvements.

Example:

```
1.0.1 → 1.0.2
```

| Category | Description |
|-----------|-------------|
| Added | New functionality or documentation |

| Fixed | Corrections and bug fixes |
| Removed | Functionality removed |
| Deprecated | Planned removal in future releases |
| Security | Security-related improvements |
