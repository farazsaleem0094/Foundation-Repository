# Welcome

Welcome to the **AI Engineering Platform (AIEP)** Foundation Repository.

This repository is not a traditional documentation repository.

It is an **Engineering Operating System** designed to build, govern, document, recover, and maintain software projects using AI-assisted engineering.

Everything inside this repository follows a Documentation-as-Code (DaC) methodology.

Documentation is the source of truth.

Implementation follows documentation—not the other way around.

---

# Purpose of this Document

This document is the official entry point into the repository.

Every human contributor and every AI session must begin here.

Its purpose is to:

- Explain how the repository is organized.
- Define the startup sequence.
- Explain repository rules.
- Explain how work is resumed.
- Point to the next required documents.
- Prevent contributors from starting in the wrong place.

# Repository Rules

The following rules are mandatory.

## Rule 1

Documentation is the source of truth.

---

## Rule 2

Architecture comes before implementation.

---

## Rule 3

Every document has:

- Permanent ID
- Version
- Status
- Dependencies
- Change history

---

## Rule 4

Every document must be understandable by beginners.

Assume zero prior knowledge.

Explain everything.

---

## Rule 5

Whenever software, repositories, packages, MCP servers, APIs, SDKs, or tools are referenced:

Always include:

- Official URL
- Complete installation instructions
- Complete commands
- Execution location
- Validation steps
- Troubleshooting guidance

---

## Rule 6

ASCII tree diagrams are not used.

Repository layouts use nested alphanumeric notation.

Example:

A)

A.1)

A.1.1)

---

## Rule 7

Every completed document must exist as its own source file.

---

## Rule 8

Every completed session updates repository state.
